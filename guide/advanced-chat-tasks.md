# 助理聊天计划任务与结果投递

助理聊天的计划任务用于把一段固定任务交给 AI 在指定时间运行。结果投递用于把任务结果发送到你的系统，例如 Webhook 或邮箱。

如果页面中看不到“计划任务”或“结果投递”，通常是管理员没有开启对应功能。管理员只关闭结果投递时，计划任务仍然可以运行，但 AI 不会获得投递工具，任务结果只会保存在会话中。

## 配置结果投递

进入 **助理聊天 > 结果投递**，点击 **新建投递**。

### Webhook 投递

选择方式为 `Webhook`，填写：

- 名称：用于在计划任务中识别这个投递目标。
- Webhook 地址：接收结果的 HTTPS 或 HTTP 地址。
- Webhook 请求头：可选，填写 JSON 或每行一个 `Header: value`。

任务完成时，平台会向 Webhook 地址发送一次 `POST` 请求。

请求头：

```http
Content-Type: application/json
```

如果你在投递配置中填写了自定义请求头，也会一起发送。

请求体格式：

```json
{
  "delivery_id": "acd_xxxxx",
  "delivery": "每日摘要 Webhook",
  "title": "每日工单摘要",
  "body": "这里是 AI 生成并要求投递的最终结果内容。",
  "format": "markdown",
  "created_at": "2026-06-28T10:30:00+08:00"
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `delivery_id` | string | 投递配置 ID。 |
| `delivery` | string | 投递配置名称。 |
| `title` | string | AI 调用投递工具时给出的结果标题。 |
| `body` | string | AI 调用投递工具时给出的结果正文。 |
| `format` | string | 可选格式，常见值为 `text`、`markdown`、`json`。 |
| `created_at` | string | 平台发起投递的时间，RFC3339 格式。 |

Webhook 返回 HTTP `2xx` 会被视为投递成功。其他状态码或网络错误会记录为投递失败，并出现在任务运行的工具调用结果中。

### 邮箱投递

选择方式为 `邮箱`，填写收件邮箱。

如果管理员允许使用系统 SMTP，你可以只填写收件邮箱；平台会使用后台统一 SMTP 发送邮件。

如果管理员不允许使用系统 SMTP，你必须在投递配置里填写自己的 SMTP：

- SMTP 主机
- SMTP 端口
- SMTP 用户名和密码（如果服务商要求）
- 发件邮箱

平台会优先使用你在投递配置中填写的 SMTP；未填写时才会在管理员允许的情况下使用系统 SMTP。

## 创建计划任务

进入 **助理聊天 > 计划任务**，点击 **新建任务**。

常用配置：

- 名称、描述：方便你在列表里识别任务。
- 智能体：可选，指定任务使用哪个智能体提示词。
- 运行时机：
  - 手动：只在你点击“运行”时执行。
  - 一次性：在指定时间执行一次。
  - 间隔：按秒数周期执行，最小间隔由平台限制。
- 会话：
  - 自动新建会话：每次任务运行时创建一个新会话。
  - 选择已有会话：把任务消息追加到已有会话上下文中。
- 自动删除会话：仅自动新建会话时可用，运行后删除本次自动创建的会话和运行记录。
- 发送的消息：真正交给 AI 执行的任务内容。
- 超时时间：任务最长运行时间，超时后会停止。
- 结果投递：选择已配置的 Webhook 或邮箱投递；不选择则只保存到会话。
- 模型、用户渠道、温度、思考强度、最大 Token：控制本次任务使用的模型和生成参数。

## AI 如何投递结果

当计划任务选择了结果投递，并且管理员开启了消息投递功能时，平台会给 AI 暴露一个名为 `deliver_result` 的工具，并提示 AI 在完成任务后调用该工具。

工具参数由 AI 生成，结构等价于：

```json
{
  "title": "结果标题",
  "body": "最终结果正文",
  "format": "markdown"
}
```

因此，你在“发送的消息”里可以明确要求结果格式，例如：

```text
统计今天新增的工单，按优先级分组，生成 Markdown 摘要。
完成后请投递结果，标题使用“今日工单摘要”。
```

如果 AI 没有调用投递工具，任务仍可能显示完成，但不会发送 Webhook 或邮件。建议在任务消息中明确写出“完成后请投递结果”。

## 查看运行结果

计划任务列表会显示最近运行状态、错误和下次运行时间。

如果任务使用已有会话或未开启自动删除会话，可以进入对应助理聊天会话查看完整上下文、AI 回复和工具调用记录。Webhook 或邮箱投递失败时，错误会出现在工具调用结果中。
