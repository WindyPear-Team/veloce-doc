# 工作室运营

工作室将多个 Agent 组织成可持续运行的协作单元。它属于创建者个人，与企业模式中的组织、成员和 RBAC 相互独立。创建或打开工作室后，从工作室详情进入“工作室运营”。

## 启用运营

首次启用时，填写运营名称、当前使命和余额运行下限。系统会为该工作室建立一份章程，并进入运行状态。

章程用于记录使命、目标、数据边界、禁止动作和审批策略。每次修改会生成新的不可变版本；后续工作按当时生效的版本执行。

## 团队与工作

在“团队”中为工作室成员绑定已有 Agent，并定义其职称。一个工作室必须且只能有一名总经理（Chief）和一名检查员（Checker）；其他成员可以承担员工、业务分析员或测试员职责。

在“工作”中创建带有明确验收条件的交付项。总经理会按优先级与并发额度安排独立内部会话。工作从计划、授权、排队到执行、复核和交付都会保留状态；遇到阻塞、可重试失败或需要人工处理时，会显示在等待决定的状态中。

每个工作项的内部会话只读，用于查看总经理安排和执行过程，不能在其中继续对话或修改记录。

## 预算与余额保护

工作室可设置每日预算、每月预算和余额运行下限：

- 每日或每月预算为 `0` 时表示不设置该项上限；设置正数后，新的工作预留额度不能超过对应限额。
- 实际模型消耗仍写入平台原有的计费账本，并回显到工作室成本汇总。
- 所有者余额达到或低于运行下限时，系统停止创建新的内部会话，将工作室标为“需要关注”，并提示补充余额或调整下限后恢复运行。

手动暂停同样会阻止新工作执行；恢复后才会继续调度。

## 运行环境与审批

工作室可绑定以下任一种运行环境：本地连接器设备（含工作目录和允许的命令前缀），或一台自己创建的托管云沙箱；两者不能同时使用。该绑定只定义工作室的运行环境，成员各自的 Agent、模型、Skill 和 MCP 配置仍单独维护。

连接器操作先由检查员判断。检查员无法安全判断的操作会暂停在原内部会话中，并出现在“审批”页；所有者可以批准或拒绝。配置连接器前，请只允许受控设备、必要的工作目录和最小化的命令前缀。

## 调度

在“调度”中设置最大并发工作数，可设范围为 `1` 到 `8`。调度器不会超过该额度同时运行工作项。

相关的托管执行环境见[托管云沙箱](/guide/advanced-chat-sandboxes)，定时触发的助理任务见[助理聊天计划任务](/guide/advanced-chat-tasks)。
