# 用量与日志

平台为每一次成功的 API 请求记录一条用量日志（TokenLog），帮助你了解消费明细。

## 在哪里看

- **用量 / 日志 页面**：逐条查看自己的请求记录；
- **数据看板（DataBoard）**：以图表形式查看用量趋势、消费分布等汇总信息；
- **仪表盘（Dashboard）**：账户总览，如余额、近期消费等。

## 一条日志包含什么

每条记录大致包含：

| 字段 | 说明 |
| --- | --- |
| 时间 | 请求发生的时间 |
| 模型 | 实际命中的模型名 |
| 渠道 / 用户渠道 | 请求最终走的上游 |
| 输入 token | 普通输入 token 数 |
| 输出 token | 生成的输出 token 数 |
| 缓存输入 token | 命中缓存的输入 token |
| 缓存写入 token | 写入缓存的 token（含 1 小时档） |
| 图像 / 音频 token | 多模态 token |
| 费用 | 本次请求的最终扣费金额 |
| API 密钥 | 本次请求使用的密钥 |

## 如何核对费用

把日志里的各维度 token 数代入 [模型与定价](/guide/models-and-pricing) 的公式即可验算：

```
费用 ≈ Σ(token 数 × 单价 / 1,000,000) × 分组倍率 × 渠道倍率
```

如有阶梯定价，单价以命中区间为准，因此实际值可能与简单线性估算略有差异。

## 用量与计费的时机

- **流式与非流式** 都会在请求完成后结算并写入日志；
- 费用是基于上游返回的真实 usage 计算的；
- 如果一次请求在上游侧失败，通常不会产生扣费（以日志记录为准）。

## 排查异常

- **费用比预期高**：检查是否命中了倍率较高的渠道/分组，或输出 token 超出预期；
- **看不到某次请求**：可能请求并未真正发往上游（如鉴权/余额/限流在网关侧就被拦截）；
- **想做对账**：导出/筛选日志，按模型或时间段汇总。
