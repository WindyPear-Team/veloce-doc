# 运行模式

在“系统管理 > 系统设置 > 基础设置”中选择平台运行模式。设置保存于系统配置，默认是运营模式；切换模式不会删除历史数据。

## 运营模式

适合面向多个用户公开运营的平台。余额、按量扣费、充值支付、签到和邀请等运营能力按常规方式工作。

## 自用模式

适合个人或小范围自用部署。系统会关闭余额扣费和支付等运营能力，避免将内部使用当作对外商业服务处理。

## 企业模式

适合单企业私有部署。启用后，系统提供企业成员、工作区（Workspace）、基于角色的访问控制（RBAC）与治理能力；切换时会回填一个企业组织，并为已有用户建立成员关系和个人工作区。

企业模式不会移除原有账户或业务记录。切换前应先确认成员权限、工作区与配额设置符合预期；模式切换不是数据隔离或数据迁移机制。

相关的企业管理接口见[控制台管理员接口](/api/console/admin)。
