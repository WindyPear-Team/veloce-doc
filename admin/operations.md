# 运行维护

本页说明管理员在“系统管理 > 系统设置”中使用的备份、日志和更新功能。运行参数与环境变量见[环境变量](/admin/configuration)。

## 配置备份与恢复

“配置备份与恢复”可将以下运行配置导出为 JSON：

- 全部系统设置；
- 用户渠道与上游渠道（包含渠道 API Key）；
- 全局模型；
- 模型价格与渠道模型映射。

导入此前导出的文件时，系统会覆盖同名的设置、渠道和模型配置，但不会删除文件中没有出现的既有记录。用户、API Key、余额、订单、日志及其他业务数据始终不在导入导出范围内。

::: warning 导出文件包含密钥
渠道配置可能包含上游 API Key。导出文件应按密钥处理，不要上传到公开仓库或通过不受信任的渠道传输。
:::

## 独立日志存储

审计、API 用量、插件和状态检查记录独立于业务主数据库，默认写入运行目录下的 `log/`。在系统设置中可以选择：

- **单个日志数据库文件**：持续写入 `log/flai-log.db`；
- **每天一个日志数据库文件**：写入 `log/flai-log-YYYY-MM-DD.db`。

可分别设置各类日志的保留时长，并按需清理全部日志。切换存储模式不会让旧文件不可读，后台会继续汇集两种布局的历史日志。

## Redis 计费协调

Redis 设置位于“系统设置 > Redis”，包括启用状态、地址、数据库编号、TLS、用户名和密码。保存后要重启服务。部署环境可用 `REDIS_URL` 或 `REDIS_*` 环境变量覆盖后台设置，完整优先级见[环境变量](/admin/configuration#redis可选)。

启用 Redis 后，系统会为同一用户的扣费事务建立分布式协调锁，并在事务提交后更新余额镜像。数据库仍是唯一账本；Redis 关闭、故障或锁等待失败时，请求会继续使用数据库的原子条件更新完成扣费。

## 自动更新

自动更新仅适用于官方 Release 二进制，支持 Linux、macOS 与 Windows 的 `amd64`、`arm64` 架构。本地 `go build` 产物的版本为 `dev`，不会参与自动更新。

在系统设置中开启自动更新并设置检查间隔（1 到 168 小时）。更新器会查询最新 Release，选择当前平台的发布资产，校验 GitHub 提供的 SHA-256 摘要后下载，再在服务退出时替换并重启二进制。

::: warning 部署要求
运行账户必须有二进制所在目录的写权限，并且部署方式必须允许进程自行重启。由容器编排、systemd 或其他进程管理器负责替换镜像/二进制的部署，不应依赖应用内自更新。
:::

发布者应推送 `v1.2.3` 这样的语义化版本标签。发布工作流会将标签写入二进制；前端仓库也必须存在同名标签。详见[构建与发布](/develop/building#github-release)。
