# 签到与邀请返佣

两项可选的运营增长功能，鼓励用户活跃与拉新。用户侧说明见 [签到与邀请](/guide/checkin-and-referral)。

## 每日签到

由后台开关控制，奖励直接进入用户余额。支持三种可叠加的奖励模式。

### 设置项

| 设置键 | 默认 | 说明 |
| --- | --- | --- |
| `每次签到奖励` | `0` | 每日固定奖励金额 |
| `签到时区` | `Asia/Shanghai` | 判定「一天」的时区 |
| `连续签到循环天数` | `7` | 连续签到周期（天） |
| `连续签到奖励 JSON` | `{}` | 各连续档位的奖励配置（JSON） |
| `随机奖励最小值` | `0` | 随机奖励下限 |
| `随机奖励最大值` | `0` | 随机奖励上限 |

### 行为

- 每个用户每天（按 签到时区 划分）最多签到一次；
- **连续签到**：连续天数累加，达到周期档位发放对应奖励；中断后连续天数重置为 1；
- **随机奖励**：在最小值与最大值间随机；
- 签到产生一条签到记录，记录奖励金额、连续天数与奖励类型。

## 邀请返佣

由返佣开关和比例控制。被邀请人消费时，按比例返佣给邀请人。

### 设置项

| 设置键 | 默认 | 说明 |
| --- | --- | --- |
| `返佣比例` | `0` | 返佣比例（如 `0.1` 表示 10%） |

### 行为

- 每个用户有唯一邀请码；
- 新用户通过邀请链接注册时，其邀请人关系在注册时确定且不可更改；
- 被邀请人每笔消费按返佣比例计算返佣，**额外**发放到邀请人余额，不影响被邀请人本次扣费；
- 每笔返佣记录为返佣日志，关联触发消费的日志。

::: tip 防刷建议
开启返佣时，结合邮箱验证、hCaptcha、IP 限制等手段降低批量注册薅羊毛的风险，见 [安全与防护](/admin/security)。返佣比例不宜过高。
:::
